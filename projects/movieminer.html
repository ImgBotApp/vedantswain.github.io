MovieMiner

Discover new movies based on your Social Network

Jan,2015 - Dec, 2015
  • View project on Github
  • View report PDF
  • Team Size: 1

Motivation

As of now audiences can discover new movies via certain methods, namely croudsourcing (ratings) or recommendation systems.

The shortcoming of most of these recommender systems as usable applications is that, it expects the user to do a lot of work (e.g. rating films or buying products) before actually giving accurate suggestions.

Solution

The solution proposed tries to deal with the first problem by harnessing the data stored in the user’s social media activity. This discards the initial setup or the initial slow-down. Social media data can be useful to calculate the similarity between users As an engineering solution, tapping into the repositories of social network sites, our system has no need of creating its own archive from scratch.

A mobile application as a client or an interface helps us further our goals of decreasing latency and increasing portability.Hence, the approach taken was to build a mobile based recommender app.

Features

  • Likes and Dislikes

    Fetching the movies liked by a user was easy to implement via fetching his or her liked movies from Facebook. However, Facebook has no user interface for users to express their dislike. Dislikes can be equally important in recommending a new movie to the user.

    Thus in order to allow this interaction with a movie, a user is enabled to modify their relation with a movie. These alterations are recorded and used for tailoring better suggestions.

  • Genre Based Browsing

    In order to provide users a method to browse movies beyond their own and those recommended to them this feature was implemented. A user can easily browse movies by selecting a genre. The user may interact with these movies in the forms of likes and dislikes as well, thus influencing future recommendations.

  • Search

    Although the browsing feature covers most of the user’s needs, the search feature is essential for two reasons. The first of these is, it makes finding a specific movie much easier. The second is, due to the crowd-sourced method of collecting movies, the list of movies that can be browsed is limited to the movies liked on Facebook by our users.

  • Detailing

    An additional screen was developed to allow users to examine movies more closely. This screen has more information about a selected movie and will help the user take a more informed decision.

Challenges

Cold Start

A very common hurdle of any recommender system is a simple paradox often termed as a cold-start problem or a chicken-egg problem.Accurate recommendations need more users. To get more users we need more accuracy.

This is a classic cyclic problem. To break free of this loop and solve the problem, MovieMiner initially shows the movies a user hasn’t liked or disliked.

Noise

The method that used to populate the repositories with movies involve querying the movie page’s name to the OMDb API. However, in certain rare cases the Facebook page name of the movie doesn’t match its IMDB title. The API uses Hamming distance against IMDB movie titles to find a match, instead of searching for the most trending among the search results. This leads to a major problem when the there exists an actual movie by the incorrect name of the page.

Existing Alternative

MovieLens, developed by GroupLens, University of Minnesota is an accurate recommender system. However it requires the user to fill in preliminary data. It doesn't use standard genres and generalises by offering clubbed combinations together. Other setbacks are that it's neither mobile nor social.

Integration for other Mediums

Since the main source of user preferences is via Facebook, other user preferences such as books, songs and games can also be learnt. This way, the similarity between users can go beyond the boundaries of the movies domain.
