Meetup

Track attendee locations for your events

Jan, 2015 - May, 2015
  • View project on Github
  • Team Size: 5

Introduction

The project aims at building up a distributed system where the nodes are mobile devices, interacting with the server. The system allows the user to set up a “meetup”, with a destination location and the time for the meetup. During the time when meetup is active, the system receives location update messages from the devices of the users involved in that particular meetup, and these location messages are sent to all the active nodes to display a their route on a map, while they arrive at the destination.

Architecture

Server Side

The server stores the user information and the meetup information. User information includes their profile, contacts and friends.

Meetup information is comprised of its details (time, location), attendees etc. Apart from which meetup information also includes the heartbeat function. This is used to take the location of a user and broadcast it to the other attendees.

Client Side

The user can also create a new meetup, by entering the details that include meetup name, meetup date and time. The user marks the destination location on a map, and chooses the contacts to be invited

Once, a meetup is active, the client sends a “HeartBeat” message to the server, indicating the device’s location after a timeout, and receives the location from all the active nodes, and displays their route on a map.

Challenges

Distribution

Handling timestamps and synchronisation over multiple devices in different timezones. This leads to different patterns being shown to every attendee based on which heartbeats they've received correctly
